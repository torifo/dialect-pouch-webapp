# dialect-pocket データ調達リサーチ: 日本語方言オープンデータ／辞書／コーパス／API

> **重要な前置き（検証ステータスについて）**
> 本調査は本来 WebSearch / WebFetch で各 URL の実在・アクセス可否を確認する前提だが、
> **実行環境でネットワークアクセス（WebSearch・WebFetch・curl）がすべて拒否された**ため、

> したがって本ドキュメントの**全エントリの URL・ライセンス・収録規模・データ形式は「未確認（unconfirmed）」**であり、
> 調査担当者（または後続の自動パイプライン）が**取り込み前に必ず一次ソースで再確認**すること。
> 記載内容は 2026-01 時点の知識に基づく既知情報であり、推測で断定した数値・ライセンスは含めず、
> 不明な項目は明示的に「不明」とした。捏造データ（生成AIによる方言の創作）は一切含まない方針。

1. **再配布可能な「まとまった方言語彙データ」は乏しい。** 権威ある NINJAL のデータ（GAJ/LAJ/COJADS 等）は機械可読でも**再配布が制限/ライセンス要確認**で、アプリでの再公開には向かない（＝**出典・検証用の参照**に留める)。

### 改訂版・取り込み戦略（実検証反映)
- **フェーズ0(地域階層)**: **CODH Geoshape(CC BY 4.0, GeoJSON)** を PostGIS/地域ツリーに投入。代替で N03。方言圏は別レイヤで手動定義。
- **フェーズ1(語彙シード)**: **Wikidata Lexeme(CC0)** + **Wiktionary(CC BY-SA)→wiktextract** から方言ラベル語を抽出。**量は中規模(数百〜)想定**、各語に出典/ライセンス/取得日の provenance 付与、CC BY-SA は継承表示。
- **フェーズ2(権威付け・差別化)**: NINJAL **GAJ/LAJ/COJADS/危機言語データ**は**出典・検証の参照**として使用（数値・分布の裏取り)。再公開はライセンス許諾が取れた範囲のみ。手動キュレーションの裏付けに最適。
- **フェーズ3(粒度)**: 自治体方言集・危機言語基礎語彙を地域単位で手動取り込み（個別ライセンス確認)。
- **UGC**: ニックネーム投稿で長期的に拡充。

## サマリ（推奨ソース上位・取り込み順の提案）

方言データは「研究機関の地図系（規模大・機械可読性低）」「自治体／市民の語彙集（粒度高・形式バラバラ）」「百科系（規模大・信頼性とライセンス要注意）」の3層に大別される。MVP では **機械可読性とライセンスの明快さ** を優先する。

**推奨取り込み順（MVP→拡張）:**

1. **行政界 GIS（国土数値情報 / e-Stat 国勢調査境界）** — 方言そのものではないが、アプリの地域階層（国 > 都道府県 > 市町村 > 方言圏）の骨格に必須。ライセンスが明快（政府標準利用規約・出典明示で商用可）で機械可読。**最初に入れる土台。**
2. **Wiktionary（日本語版）の方言ラベル付き項目 / Wikidata** — CC BY-SA（Wikidata は CC0）。語数が多く API・ダンプで機械可読。出典明示＋同一ライセンス継承の条件付きだが MVP のシード語彙として有力。**ただし信頼性は項目差が大きいため出典として要素ごとに原典リンクを保持。**
3. **NINJAL（国立国語研究所）の方言地図・データベース（LAJ / GAJ / 方言談話資料 等）** — 一次研究データとして信頼性最高。**ただしライセンスと機械可読形式が課題**（多くが画像・PDF・閲覧系DBで一括ダウンロード不可の可能性）。出典の「核」として位置づけ、ライセンス確認後に段階導入。

**結論:** 「GIS で地域階層を作る → Wiktionary/Wikidata で語彙を量的にシード → NINJAL 系で権威ある出典を裏付け」の3段構成が MVP として現実的。自治体語彙集は粒度が高いが取り込みコストが高いため第2フェーズ。

## 初期シード（MVP）に使える具体的な組み合わせの提案

**フェーズ0（地域階層の骨格）**
- **国土数値情報 行政区域データ**（または e-Stat 境界データ）を PostGIS に投入し、`国 > 都道府県 > 市区町村` のポリゴン階層を構築。
- 方言圏（例: 東日本／西日本、九州、東北、関西 等）は行政界の上位に「方言区画」レイヤを手動で対応付け（NINJAL や Wikipedia の方言区画記事を区画定義の出典として参照）。

**フェーズ1（語彙シード）**
- **Wiktionary 日本語版**から「方言」ラベル付き見出し語を API/ダンプで抽出 → 意味・用例・読み・地域ラベルを取得。
- 各語に **provenance（出典URL・ライセンス CC BY-SA・取得日）** を必ず付与。ライセンス継承（同一条件公開）をアプリ表示／配布条件に組み込む。
- 地域ラベルはフェーズ0の地域階層へマッピング（曖昧な場合は「都道府県不明・方言圏のみ」として粒度を落として保持）。

**フェーズ2（権威ある裏付けと拡張）**
- **NINJAL（LAJ/GAJ/方言DB）** を出典として、シード語の分布・信頼性を裏付け（ライセンス確認後）。音声付きデータがあれば発音機能に活用。
- **自治体の方言集**を地域単位で順次取り込み（個別ライセンス確認・PDF/HTML抽出）。粒度が最も高く、アプリの差別化要素になる。

**provenance 設計（全フェーズ共通・プロジェクト方針の中核）**
- 各エントリに最低限: `source_name` / `source_url` / `license` / `retrieved_at` / `original_reference`（一次出典があれば）を保持。
- ライセンスが「不明」のものは公開フラグを `false`（内部レビュー待ち）にし、確認できるまでユーザーに出さない。
